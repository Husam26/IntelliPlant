1. Executive Summary

IntelliPlant is a full-stack, AI-powered Industrial Knowledge Intelligence Platform that transforms unstructured facility documentation — equipment manuals, P&IDs, maintenance logs, compliance reports, and safety SOPs — into a unified, queryable knowledge layer.

At its core, IntelliPlant combines Retrieval-Augmented Generation (RAG) with a real-time Digital Twin, enabling plant engineers to ask natural language questions and receive precise, citation-backed answers grounded in their actual facility documentation — while simultaneously visualizing equipment health, tracking compliance, and predicting failures.

2. Problem Statement & Motivation

The Challenge

Modern industrial facilities generate and rely on vast quantities of unstructured documentation:

  • Equipment Manuals — OEM-supplied maintenance procedures, torque specifications, part numbers
  • P&IDs (Piping & Instrumentation Diagrams) — Complex schematic drawings linking physical assets
  • SCADA Telemetry Logs — Real-time sensor data from temperature, pressure, vibration sensors
  • Maintenance Records — Historical work orders, failure analyses (FMEA), corrective actions
  • Compliance Reports — OSHA safety logs, ISO 55000 audits, API 653 inspections, EPA emissions checks

Current Pain Points

  • 60%+ of engineer time is spent manually searching scattered file shares and physical binders
  • Critical maintenance procedures are buried in 200+ page PDF manuals
  • No single system connects document knowledge to live equipment telemetry
  • Compliance gaps are only discovered during expensive audit cycles
  • Failure patterns repeat because historical incident data is siloed

Impact of the Problem

  • Unplanned downtime costs industrial plants $250,000/hour on average
  • Compliance violations result in regulatory fines and operational shutdowns
  • Knowledge loss from retiring workforce creates institutional memory gaps

3. Proposed Solution

IntelliPlant addresses these challenges by creating an AI-first knowledge platform that:

  1. Ingests all facility documentation (PDF, DOCX, XLSX, TXT) through a drag-and-drop interface
  2. Indexes content into a FAISS vector store using sentence-transformer embeddings for semantic search
  3. Extracts structured entities (equipment IDs, asset tags, failure modes) using LLM-powered Named Entity Recognition
  4. Answers natural language questions via a RAG pipeline that retrieves relevant context and generates grounded, cited responses
  5. Visualizes the entire plant as an interactive Digital Twin where users can add, connect, and monitor assets in real-time
  6. Predicts failure patterns by analyzing incident history and correlating with equipment telemetry
  7. Tracks regulatory compliance automatically, flagging overdue and non-compliant items

7. Data Flow & RAG Pipeline

7.1 Document Ingestion Pipeline

  1. Upload: User uploads PDF/DOCX/XLSX via the Documents page
  2. Text Extraction: document_processor.py extracts raw text using PyPDF2/python-docx
  3. Chunking: Text is split into overlapping chunks (1000 chars, 200 overlap) for optimal retrieval
  4. Embedding: Each chunk is converted to a 384-dimensional vector using all-MiniLM-L6-v2
  5. FAISS Indexing: Vectors are added to the FAISS index for similarity search
  6. Entity Extraction: LLM extracts equipment IDs, asset names, and categories via NER
  7. AI Summary: LLM generates a concise summary of the entire document
  8. Database Storage: Metadata, entities, and chunk references stored in SQLite

7.2 RAG Query Pipeline (AI Copilot)

  1. Query Embedding: User's question is embedded using the same sentence-transformer model
  2. Vector Retrieval: FAISS performs top-K similarity search (k=5) across all indexed chunks
  3. Context Assembly: Retrieved chunks are assembled into a structured context window with source metadata
  4. LLM Generation: Groq (LLaMA 3.3 70B) generates a grounded answer using the retrieved context
  5. Citation Extraction: Source document names, sections, and excerpts are attached as citations
  6. Fallback Engine: If the backend is unavailable, the client-side Industrial Knowledge Base provides pre-indexed answers
Key Design Decision: We use FAISS-CPU (not GPU) to ensure the platform runs on any standard laptop without requiring specialized hardware — making it accessible for hackathon demonstrations and on-premise industrial deployments.

11. Digital Twin — Design & Implementation

The Digital Twin is IntelliPlant's most visually impressive and technically complex feature. It provides a fully interactive, database-backed 2D plant visualization.

Key Capabilities

  • Add Assets: Modal form with asset ID, name, type, criticality, location, health score, and connection picker
  • Drag to Reposition: 5px movement threshold distinguishes clicks from drags; positions saved to DB on mouseUp
  • Process Flow Lines: Animated SVG dashed lines between connected assets with health-based coloring
  • Connection Editor: Click any asset → detail panel shows toggle chips to connect/disconnect from other assets in real-time
  • Health Visualization: Green (≥80%), Amber (60-79%), Red (<60%) with pulsing glow animations on critical assets
  • Delete Assets: Red delete button in the detail panel with confirmation dialog
  • Empty State: New clients see a blank canvas with a clear "Add Asset" call-to-action

Technical Implementation

  • Asset positions stored as x_pos, y_pos (0-100% normalized coordinates) in the assets table
  • Connections stored as comma-separated asset IDs in the connections column
  • Drag uses useRef (not useState) for immediate access without React batching delays
  • SVG layer renders connection lines; HTML layer renders asset nodes — both use percentage-based positioning

12. AI Copilot — RAG Engine Deep Dive

Architecture

The AI Copilot implements a multi-layer intelligence pipeline:

  1. Embedding Layer: sentence-transformers/all-MiniLM-L6-v2 generates 384-dim dense vectors
  2. Retrieval Layer: FAISS IndexFlatL2 performs exact nearest neighbor search
  3. Context Layer: Top-5 retrieved chunks are assembled with source metadata
  4. Generation Layer: Groq API (LLaMA 3.3 70B Versatile) generates grounded answers
  5. Citation Layer: Source documents, sections, and excerpts are extracted and returned

Fallback Architecture

IntelliPlant implements a three-tier fallback system to ensure zero downtime:

  1. Primary: Full RAG pipeline (FAISS + Groq LLM)
  2. Secondary: Backend Industrial Knowledge Base (keyword-matched pre-indexed answers)
  3. Tertiary: Client-side fallback engine in Copilot.jsx (works even when backend is offline)

13. Setup & Installation Guide

Prerequisites

  • Python 3.12+ with pip
  • Node.js 18+ with npm
  • Groq API Key (free at console.groq.com)

Step-by-Step Installation

1. Clone Repository

git clone https://github.com/Husam26/IntelliPlant.git
cd IntelliPlant

2. Setup Backend

cd backend
python -m venv venv
venv\Scripts\activate          # Windows
pip install -r requirements.txt

3. Configure Environment

# Create .env in project root
GROQ_API_KEY=your_groq_api_key_here

4. Seed Demo Data

python seed_demo_data.py

5. Start Backend

uvicorn app.main:app --reload
# Backend: http://localhost:8000
# Swagger: http://localhost:8000/docs

6. Start Frontend

cd frontend
npm install
npm run dev
# Frontend: http://localhost:5173
